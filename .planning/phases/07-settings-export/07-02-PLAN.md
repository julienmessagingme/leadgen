**useSettings.js hooks** (follow useLeads.js pattern with api client from AuthContext):

- `useIcpRules()` -- useQuery GET `/api/settings/icp-rules`, queryKey `["icp-rules"]`
- `useCreateIcpRule()` -- useMutation POST `/api/settings/icp-rules`, invalidates `["icp-rules"]`
- `useUpdateIcpRule()` -- useMutation PUT `/api/settings/icp-rules/:id`, invalidates `["icp-rules"]`
- `useDeleteIcpRule()` -- useMutation DELETE `/api/settings/icp-rules/:id`, invalidates `["icp-rules"]`
- `useSuppression()` -- useQuery GET `/api/settings/suppression`, queryKey `["suppression"]`
- `useAddSuppression()` -- useMutation POST `/api/settings/suppression`, invalidates `["suppression"]`
- `useDeleteSuppression()` -- useMutation DELETE `/api/settings/suppression/:hash`, invalidates `["suppression"]`
- `useConfig()` -- useQuery GET `/api/settings/config`, queryKey `["config"]`
- `useUpdateConfig()` -- useMutation PATCH `/api/settings/config/:key`, invalidates `["config"]`
- `useWatchlist()` -- useQuery GET `/api/settings/watchlist`, queryKey `["watchlist"]`
- `useCreateWatchlistEntry()` -- useMutation POST `/api/settings/watchlist`, invalidates `["watchlist"]`
- `useUpdateWatchlistEntry()` -- useMutation PUT `/api/settings/watchlist/:id`, invalidates `["watchlist"]`
- `useDeleteWatchlistEntry()` -- useMutation DELETE `/api/settings/watchlist/:id`, invalidates `["watchlist"]`
- `useCronSchedule()` -- useQuery GET `/api/settings/cron`, queryKey `["cron"]`
- `useExportLeads()` -- function that builds URL with query params and triggers `window.open(url)` for download (or use fetch + blob download pattern with auth header)
