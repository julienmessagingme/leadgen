Replace JavaScript-side aggregation in dashboard endpoints with PostgreSQL RPC functions called via supabase.rpc(), eliminating full table scans and N+1 patterns.

Purpose: Dashboard currently fetches ALL leads into Node.js memory and computes aggregates in JS loops. Cron status makes 6 sequential queries. These are the most frequent API calls (every dashboard page load) and must be optimized.
Output: 3 SQL migration files with RPC functions + refactored dashboard.js using supabase.rpc() calls.
</objective>
