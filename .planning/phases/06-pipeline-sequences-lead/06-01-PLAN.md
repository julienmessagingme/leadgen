  <name>Task 1: Create leads API router with list, detail, action, and bulk-action endpoints</name>
  <files>src/api/leads.js, src/index.js</files>
  <action>
Create `src/api/leads.js` as an Express Router with authMiddleware applied at router level (same pattern as `src/api/dashboard.js`).

**GET /** -- List leads with server-side filtering and pagination via Supabase query builder:
- Query params: `status` (comma-separated), `tier`, `source` (signal_category), `search` (name or company ilike), `sort` (default: icp_score), `order` (default: desc), `limit` (default 200, max 500), `offset` (default 0), `paused` (optional: "true"/"false" to filter by metadata.is_paused)
- Always use `select("*", { count: "exact" })` for total count
- Exclude `status = "disqualified"` by default unless status param explicitly includes it
- Search uses `.or('full_name.ilike.%term%,company_name.ilike.%term%')` -- sanitize search term to remove PostgREST operators (periods, commas)
- Sort validation: only allow `icp_score`, `created_at`, `signal_date`, `status`
- Return `{ leads: data, total: count }`

**GET /:id** -- Single lead detail:
- Fetch lead by ID with `.select("*").eq("id", id).single()`
- Return 404 if not found
- Return full lead object

**PATCH /:id/action** -- Individual lead action (pause/resume/exclude):
- Accept `{ action: "pause" | "resume" | "exclude" }` in body
- **pause:** Set `metadata.is_paused = true` and `metadata.paused_at` timestamp. Do NOT change status (preserves pipeline position per RESEARCH.md Pitfall 1).
- **resume:** Remove `metadata.is_paused` and `metadata.paused_at` from metadata. Do NOT change status.
- **exclude:** Set status to `"disqualified"`, add `excluded_at` and `excluded_reason: "manual_rgpd"` to metadata. Insert SHA-256 hashes of email and linkedin_url into `suppression_list` table with upsert on `hashed_value`. Use `require("crypto")`.
- Return `{ ok: true, action: "paused"|"resumed"|"excluded" }`

**POST /bulk-action** -- Bulk lead actions:
- Accept `{ ids: number[], action: "pause"|"resume"|"exclude" }` in body
- Validate: ids must be array, length 1-100
- For each action type, fetch leads by IDs then apply same logic as individual action
- Return `{ ok: true, processed: N }`

Mount in `src/index.js`: add `app.use("/api/leads", require("./api/leads"));` after the dashboard router mount line.

Use CommonJS (require/module.exports) to match existing backend pattern.
