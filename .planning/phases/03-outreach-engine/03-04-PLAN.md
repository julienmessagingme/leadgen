**Task E -- WhatsApp template creation (replace placeholder task-e-whatsapp.js):**
Module exports `async function taskEWhatsapp(runId)`.

**Lead selection:**
Query Supabase leads where:
- email_sent_at <= now() - 7 days (J+14 total: 7 days after invitation + 7 more after email)
  OR invitation_sent_at <= now() - 14 days (direct J+14 from invitation if no email was sent)
- status IN ('email_sent', 'invitation_sent') -- leads that did not reply/connect
- whatsapp_template_created_at IS NULL (not already in WhatsApp pipeline)
- lead has a phone number (phone IS NOT NULL) -- from FullEnrich or enrichment
- tier IN ('hot', 'warm')

**For each selected lead:**
1. Check suppression list (email + linkedin_url).
2. **WA-05:** Call `generateWhatsAppBody(lead)` from message-generator.js. If null, skip.
3. **WA-01:** Create template name: `leadgen_${lead.id.substring(0, 8)}_${Date.now()}` (unique per lead).
4. Call `createWhatsAppTemplate(templateName, body, process.env.CALENDLY_URL)` from messagingme.js.
5. Update lead metadata: store template_name, template_created_at, template_status='pending'.
6. Update lead: set whatsapp_template_created_at = now().
7. Log success.
8. Per-lead error isolation.

**WhatsApp Polling (replace placeholder whatsapp-poll.js):**
Module exports `async function whatsappPoll(runId)`.

**WA-02:** Runs every 15 min (Mon-Fri 9h-18h, already configured in scheduler).

**Flow:**
1. Query Supabase leads where metadata->template_status = 'pending' AND whatsapp_template_created_at IS NOT NULL.
2. Call `syncTemplates()` from messagingme.js to refresh Meta statuses.
3. For each pending lead:
   a. Call `listTemplates(lead.metadata.template_name)` from messagingme.js.
   b. Parse status from response.
   c. If APPROVED:
      - **WA-03:** Call `sendWhatsAppByUserId(lead.phone, templateNamespace, templateName, 'fr', {})` from messagingme.js. Get namespace from the template list response.
      - Update lead: status='whatsapp_sent', whatsapp_sent_at=now(), metadata.template_status='approved'.
      - Log success.
   d. If REJECTED:
      - **WA-04:** Alert Julien via `sendWhatsAppByUserId(process.env.JULIEN_WHATSAPP_PHONE, namespace, 'daily_leadgen_briefing', 'fr', { body: ['ALERTE: Template WhatsApp rejete pour ' + lead.full_name + '. Raison possible: contenu non conforme Meta. Lead ID: ' + lead.id] })`.
      - Note: This uses a pre-approved utility template. If that template does not exist yet, fall back to logging the alert. The utility template 'daily_leadgen_briefing' should be created manually once with a single {{1}} body parameter.
      - Update metadata: template_status='rejected'. Log warning.
   e. If still PENDING:
      - Check if created > 24h ago. If yes: **WA-04** timeout alert to Julien (same mechanism as rejection alert). Update metadata: template_status='timeout'. Log warning.
      - If < 24h: do nothing, will check again next poll.
4. Per-lead error isolation.

**Imports:** supabase, messagingme (createWhatsAppTemplate, listTemplates, syncTemplates, sendWhatsAppByUserId), suppression (isSuppressed), message-generator (generateWhatsAppBody), logger (log).

**INMAIL-01: Lead selection:**
Query Supabase leads where:
- icp_score >= 80
- status IN ('prospected', 'invitation_sent', 'new', 'enriched', 'scored') -- leads Julien might want to InMail manually
- Order by icp_score DESC
- Limit 3

If no leads found with score >= 80, log "no hot leads for briefing" and return.

**INMAIL-02: Generate InMail for each lead:**
For each of the top 3 leads:
- Call `generateInMail(lead)` from message-generator.js.
- Returns `{ subject, body }`. If null, skip this lead in briefing.

**Format briefing text:**
Build a formatted text message for Julien:
```
BRIEFING INMAIL DU [date]

1. [lead.full_name] - [lead.headline]
   Entreprise: [lead.company_name]
   Score: [lead.icp_score] | Signal: [lead.signal_source]

   Objet: [inmail.subject]
   ---
   [inmail.body]
   ---
   Profil: [lead.linkedin_url]

2. [next lead...]
```

**INMAIL-03: Send briefing to Julien via WhatsApp:**
- Call `sendWhatsAppByUserId(process.env.JULIEN_WHATSAPP_PHONE, namespace, 'daily_leadgen_briefing', 'fr', { body: [briefingText] })`.
- The 'daily_leadgen_briefing' template must be pre-approved with a single {{1}} body parameter. Store namespace in MESSAGINGME_TEMPLATE_NAMESPACE env var.
- If the WhatsApp send fails (template not found, API error), fall back to logging the full briefing text to Supabase logs so Julien can still access it.
- Log success or fallback.

**Edge cases:**
- If all 3 generateInMail calls return null, log "all InMail generations failed" and skip WhatsApp send.
- If JULIEN_WHATSAPP_PHONE is not set, log error and return (do not crash).

**Imports:** supabase, messagingme (sendWhatsAppByUserId), message-generator (generateInMail), logger (log).
