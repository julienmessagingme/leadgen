Replace the placeholder task-d-email.js on VPS with full implementation. Module exports `async function taskDEmail(runId)`.

**Lead selection:**
Query Supabase leads where:
- status = 'invitation_sent' (invitation sent but not connected)
- invitation_sent_at <= now() - 7 days (J+7 delay elapsed)
- email_sent_at IS NULL (not already emailed)
- tier IN ('hot', 'warm')
Order by icp_score DESC.

**For each selected lead, execute 4 pre-checks in order. If ANY check fails, skip to next lead:**

**Step 1 -- EMAIL-01: FullEnrich email enrichment**
- If lead already has a verified email (email IS NOT NULL AND email_verified = true in metadata), skip enrichment.
- Otherwise call `enrichContactInfo(lead.linkedin_url)` from fullenrich.js (existing module from Phase 2).
- Filter result: only accept confidence 'high' or 'medium'. If low or null, log and skip lead.
- Update lead email field and set metadata.email_verified = true, metadata.email_source = 'fullenrich'.

**Step 2 -- EMAIL-02: HubSpot email dedup**
- Call `existsInHubspotByEmail(lead.email)` from hubspot.js (added in plan 03-01).
- If found in HubSpot, log "lead email exists in HubSpot, skipping" and skip lead.

**Step 3 -- EMAIL-03: LinkedIn inbox reply check**
- Call `searchInbox(lead.full_name)` from bereach.js (added in plan 03-01).
- Parse response: if any conversation found with this lead where they sent a message (a reply), update lead status to 'replied' and skip.
- This is best-effort: false negatives are acceptable (see research pitfall 5). Search by full_name or first_name + last_name.

**Step 4 -- EMAIL-04: Suppression list check**
- Call `isSuppressed(lead.email, lead.linkedin_url)` from suppression.js.
- If suppressed, log and skip.

**After all 4 checks pass:**

**Step 5 -- EMAIL-05: Generate email**
- Call `generateEmail(lead)` from message-generator.js.
- Returns `{ subject, body }`. If null (generation failed), skip lead.

**Step 6 -- EMAIL-06: Send via Gmail SMTP**
- Call `sendEmail(lead.email, subject, body)` from gmail.js.
- Update lead: set email_sent_at = now(), status = 'email_sent', store subject and messageId in metadata.
- Log success.

**Rate limiting:** Add 5-10s delay between emails (`5000 + Math.random() * 5000` ms) to avoid Gmail rate limits. Use sleep() from bereach.js.

**Error isolation:** Per-lead try/catch. One FullEnrich timeout or Gmail error should not crash the batch.

**Log summary:** X emails sent, Y skipped (with breakdown: Z no email, W in HubSpot, V replied, U suppressed).

**Imports:** supabase, enrichContactInfo (from fullenrich), existsInHubspotByEmail (from hubspot), searchInbox/sleep (from bereach), isSuppressed (from suppression), generateEmail (from message-generator), sendEmail (from gmail), log (from logger).
