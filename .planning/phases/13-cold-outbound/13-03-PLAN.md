  <action>
Create `src/lib/cold-outbound-pipeline.js` (CommonJS module):

**Export:** `executeColdSearch(searchId, filters, runId)`

**Pipeline steps (modeled after task-a-signals.js pattern):**

1. **Update search status to 'running':**
   ```js
   await supabase.from('cold_searches').update({ status: 'running' }).eq('id', searchId);
   ```

2. **Scrape Sales Nav:**
   - Call `searchSalesNav(filters, runId)` from sales-nav-scraper.js
   - Update cold_searches.leads_found = profiles.length
   - If 0 profiles: set status 'completed', leads_found 0, return early

3. **Dedup against existing leads:**
   - For each profile, check if linkedin_url already exists in leads table
   - Skip duplicates (log count)

4. **Enrich each lead (sequential, with humanDelay between):**
   For each profile (not already in leads):
   a. Try to get email from enrichment.js enrichLead (BeReach profile visit may include email)
      - IMPORTANT: Cold leads may not have a prior signal, so construct a minimal signal object: { linkedin_url, first_name, last_name, headline, company_name }
   b. If no email found, try FullEnrich: enrichContactInfo(linkedin_url, runId)
   c. If still no email: mark lead metadata.email_status = 'no_email' (keep lead per CONTEXT decision)
   d. Update cold_searches.leads_enriched incrementally after each lead

5. **Score ICP for cold leads:**
   - Load ICP rules via loadIcpRules()
   - For each enriched lead: scoreLead(lead, [], rules)
     - Pass empty newsEvidence array (cold leads have no news signal)
     - The scorer already handles missing data gracefully
   - NOTE per CONTEXT: "Scoring ICP specifique pour les leads cold (poids differents des leads signaux)" — add a cold_outbound flag to the lead object so the scorer can detect it. If the scorer does not currently handle cold differently, that's OK for now — the existing scoring will work and can be tuned later via ICP rules in settings.

6. **Insert leads into pipeline:**
   - For each scored lead, upsert into `leads` table:
     - linkedin_url (primary dedup key)
     - first_name, last_name, headline, company_name
     - email (if found)
     - icp_score, icp_tier (from scorer)
     - status: 'new'
     - signal_type: 'cold_search'
     - signal_category: 'cold_outbound'
     - metadata: { search_id: searchId, source_origin: 'sales_nav', email_status: 'found'|'no_email', cold_outbound: true }
   - Use upsert on linkedin_url conflict: update icp_score/tier if new score is higher

7. **Finalize search record:**
   - Update cold_searches: status 'completed', leads_found, leads_enriched, completed_at = now()
   - On error: status 'error', error_message = err.message

8. **Send completion email to Julien:**
   - Use gmail.js sendEmail to GMAIL_USER
   - Subject: "Recherche cold terminee - {leads_found} leads trouves"
   - Body: summary with filters used, leads found, leads enriched, leads with email, search ID
   - On error: include error_message in email

**Error isolation pattern (from task-a):**
- Wrap each lead enrichment in try/catch — one failure does not crash batch
- Always update cold_searches status in finally block
- Log each step with runId to Supabase logs table
