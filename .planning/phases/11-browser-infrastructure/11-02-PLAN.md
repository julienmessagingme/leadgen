<task type="auto">
  <name>Task 1: Rate limiter et delais humains dans browser.js</name>
  <files>src/lib/browser.js</files>
  <action>
Enrichir le module `src/lib/browser.js` cree en plan 01 avec :

1. **Compteur de pages global** :
   - Variable module-level : `let pageCount = 0; let pageCountDate = null;`
   - Fonction interne `resetDailyCounterIfNeeded()` : compare la date actuelle (Europe/Paris timezone, format YYYY-MM-DD) avec `pageCountDate`. Si differente, reset `pageCount = 0` et met a jour `pageCountDate`. Utiliser le meme pattern que getTodayStartParis dans task-a-signals.js (toLocaleDateString avec timeZone Europe/Paris).

2. **`navigateWithLimits(page, url, options = {})`** :
   - Appelle `resetDailyCounterIfNeeded()`
   - Verifie `pageCount < 100`. Si >= 100, log un warning via logger "Daily page limit reached (100/day) - navigation blocked" et throw une Error.
   - Appelle `humanDelay()` (delai AVANT la navigation)
   - Navigue : `await page.goto(url, { waitUntil: 'domcontentloaded', timeout: 30000, ...options })`
   - Incremente `pageCount++`
   - Log au niveau debug : `"Page ${pageCount}/100: ${url}"`
   - Retourne la response de page.goto

3. **`humanDelay(minMs = 3000, maxMs = 8000)`** :
   - Calcule un delai aleatoire entre minMs et maxMs : `Math.floor(Math.random() * (maxMs - minMs)) + minMs`
   - Attend ce delai avec `await new Promise(resolve => setTimeout(resolve, delay))`
   - Log au niveau debug : `"Human delay: ${delay}ms"`
   - Exporter la fonction pour usage dans les phases suivantes (les scrapers auront besoin de delais entre actions sur une meme page)

4. **`getPageCount()`** : Retourne `{ count: pageCount, limit: 100, date: pageCountDate }` pour monitoring.

5. **Exports mis a jour** : Ajouter `navigateWithLimits`, `humanDelay`, `getPageCount` aux module.exports existants. Ne pas casser les exports existants (createBrowserContext, closeBrowser, validateSession).

Important : le compteur est en memoire (pas en base). Si le process redarre, le compteur repart a 0. C'est acceptable pour le volume actuel (<100/jour). Les phases suivantes utiliseront `navigateWithLimits` au lieu de `page.goto` directement.
  </action>
