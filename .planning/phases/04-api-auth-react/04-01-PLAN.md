Create `src/api/auth.js` -- Express Router with:
- POST `/login`: accepts `{ email, password }` JSON body. Compares email against `process.env.DASHBOARD_USER`, password against `process.env.DASHBOARD_PASSWORD_HASH` using `bcryptjs.compare()`. On match, signs JWT with `process.env.JWT_SECRET` and `{ expiresIn: '7d' }`, returns `{ token }`. On mismatch, returns 401 `{ error: 'Invalid credentials' }`. Use consistent error message for both wrong email and wrong password (no user enumeration).
- GET `/check`: a trivial endpoint behind auth middleware that returns `{ ok: true }` -- used by React to validate stored token on mount.

Create `src/api/middleware.js` -- exports a function `authMiddleware(req, res, next)`:
- Reads `Authorization` header. If missing or not `Bearer ...`, returns 401.
- `jwt.verify(token, process.env.JWT_SECRET)` -- on success, sets `req.user = decoded` and calls `next()`. On error (expired, invalid), returns 401.

Create `frontend/src/api/client.js` (ESM -- frontend is ESM):
- Base URL `/api`. Single `request(method, path, body)` function.
- Reads token from `localStorage.getItem('token')`.
- Attaches `Authorization: Bearer <token>` if token exists.
- On 401 response: `localStorage.removeItem('token')`, `window.location.href = '/login'`.
- On non-ok response: throw Error with message from response body.
- Export `api` object with `get`, `post`, `patch`, `delete` methods.

Create `frontend/src/context/AuthContext.jsx`:
- `AuthProvider` component wrapping children in context.
- State: `token` (init from localStorage), `isLoading` (init true).
- On mount (useEffect): if token exists, call `api.get('/auth/check')`. On success, set isLoading false. On error, remove token from localStorage, set token null, set isLoading false. If no token, just set isLoading false.
- `login(email, password)`: calls `api.post('/auth/login', { email, password })`, stores token in localStorage, sets state.
- `logout()`: removes token from localStorage, sets token null.
- Export `useAuth` hook.
