  <name>Task 1: Create signal collector module</name>
  <files>src/lib/signal-collector.js</files>
  <action>
    Create `src/lib/signal-collector.js`:
    - Import: supabase, bereach (all collect/search functions), logger

    Export `collectSignals(runId)`:
    1. Load active watchlist entries: `supabase.from('watchlist').select('*').eq('is_active', true)`
    2. Initialize empty signals array
    3. For each watchlist source, dispatch by source_type:

    **source_type = 'competitor_page' (SIG-01):**
    - Call bereach to get recent posts from the competitor page (source.source_url)
    - For each of the last 3 posts:
      - Collect likers via bereach.collectPostLikers(post.url)
      - Collect commenters via bereach.collectPostCommenters(post.url)
      - Format each as signal with signal_type='like'/'comment', signal_category='concurrent'
    - Add delay (1-3 seconds) between BeReach calls to respect rate limits

    **source_type = 'influencer' (SIG-03):**
    - Same pattern as competitor_page but signal_category='influenceur'

    **source_type = 'keyword' (SIG-02):**
    - Call bereach.searchPostsByKeywords(source.keywords)
    - Extract post authors as leads
    - Format with signal_type='post', signal_category='sujet'

    **source_type = 'job_keyword' (SIG-04):**
    - Call bereach.searchJobs(source.keywords)
    - For each job result, extract the hiring company
    - TODO: search for CX/Digital decision-makers at that company (may need additional BeReach search)
    - Format with signal_type='job', signal_category='job'

    Internal `formatSignals(profiles, signalType, signalCategory, source)`:
    - Map each profile to: { linkedin_url, first_name, last_name, headline, signal_type, signal_category, signal_source: source.source_label, signal_date: new Date().toISOString(), sequence_id: source.sequence_id }
    - Handle various BeReach response field names (profileUrl vs url, firstName vs name split)

    Internal `sleep(ms)` for rate limiting delays.

    Error isolation: Each source is wrapped in try/catch. One failing source does not crash the collection.
    Log: Number of signals collected per source, total signals.
  </action>

<task type="auto">
  <name>Task 2: Replace task-a placeholder with full signal pipeline</name>
  <files>src/tasks/task-a-signals.js</files>
  <action>
    Replace the existing placeholder in `src/tasks/task-a-signals.js` with the full pipeline.

    The task-a function receives `runId` from the scheduler's registerTask wrapper.

    **Pipeline steps:**

    1. **Check BeReach limits** — call bereach.checkLimits(). Log remaining quota. If critically low, log warning but continue (operator decides).

    2. **Check daily lead count (SIG-08)** — count leads created today:
       `supabase.from('leads').select('id', { count: 'exact', head: true }).gte('created_at', todayStart)`
       where todayStart = today 00:00 Europe/Paris in ISO format.
       If count >= 50: log "Daily lead limit reached (50), skipping", return early.
       Calculate remaining = 50 - count.

    3. **Collect signals** — call collectSignals(runId). Log total collected.

    4. **Dedup** — call dedup(rawSignals, runId). Log count after dedup.

    5. **Limit to remaining quota** — slice deduped signals to `remaining` count.

    6. **Load ICP rules** — call loadIcpRules() once for the entire batch.

    7. **Process each signal** (sequential, with 1-3s delay between):
       a. enrichLead(signal, runId) — profile + company + Sales Nav
       b. gatherNewsEvidence(enrichedLead, runId) — company news
       c. scoreLead(enrichedLead, newsEvidence, rules, runId) — ICP scoring
       d. If tier === 'cold': log skip, continue
       e. Insert hot/warm lead into Supabase leads table:
          - Include all enriched fields + icp_score + tier + signal metadata
          - Set status = 'new', linkedin_url_canonical from dedup step
       f. Log inserted lead (tier, name, score)

    8. **Summary log** — total signals collected, after dedup, processed, inserted (hot/warm), skipped (cold).

    Error handling:
    - Each individual lead processing is wrapped in try/catch — one lead failing does not crash the batch
    - Log errors with lead's linkedin_url for debugging
    - Overall task-a is already wrapped by registerTask (from Phase 1 scheduler) but add its own top-level try/catch for the pipeline steps

    Import all modules: signal-collector, dedup, enrichment, news-evidence, icp-scorer, bereach (checkLimits), supabase, logger.
