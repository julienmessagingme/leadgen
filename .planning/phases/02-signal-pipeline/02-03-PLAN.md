  <name>Task 1: Create Anthropic client singleton and ICP scorer module</name>
  <files>src/lib/anthropic.js, src/lib/icp-scorer.js</files>
  <action>
    Install dependency on VPS: `npm install @anthropic-ai/sdk` in /home/openclaw/leadgen/.

    Create `src/lib/anthropic.js`:
    - Import Anthropic from @anthropic-ai/sdk
    - Create and export singleton: `new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })`
    - Simple module, just the client instance

    Create `src/lib/icp-scorer.js`:
    - Import: anthropic (client), supabase, logger

    Export `loadIcpRules()`:
    - Query supabase.from('icp_rules').select('*')
    - Return the rules array
    - Called once per task-a run, results passed to scoreLead

    Export `scoreLead(lead, newsEvidence, rules, runId)`:

    **Step 1: Freshness TTL check (ICP-04)**
    - Read freshness rules from rules array (category='freshness')
    - Calculate signal age in days from lead.signal_date
    - If age > skip_days (default 15): return { ...lead, tier: 'cold', icp_score: 0 } immediately
    - Track freshness malus: age > malus_days (10) = -15, age > warn_days (5) = -5, else 0

    **Step 2: Claude Haiku scoring (ICP-01)**
    - Build structured prompt with lead data + ICP rules (titles, sectors, sizes, seniority from rules)
    - Prompt in French (matching the business context)
    - Include: lead name, headline, company, size, sector, location, news summaries
    - Call anthropic.messages.create with:
      - model: 'claude-haiku-4-5'
      - max_tokens: 512
      - messages: [{ role: 'user', content: prompt }]
      - output_config: { format: { type: 'json_schema', schema: { type: 'object', properties: { icp_score: { type: 'number' }, tier: { type: 'string', enum: ['hot', 'warm', 'cold'] }, reasoning: { type: 'string' } }, required: ['icp_score', 'tier', 'reasoning'], additionalProperties: false } } }
    - Parse response: JSON.parse(response.content[0].text)

    **Step 3: Signal weight bonus (ICP-03)**
    - Read signal_weights from rules (category='signal_weights')
    - Apply bonus based on lead.signal_category: concurrent +25, influenceur +15, sujet +10, job +5
    - These are deterministic post-processing, NOT in the Claude prompt

    **Step 4: Freshness malus (ICP-04)**
    - Apply the malus calculated in Step 1

    **Step 5: News bonus (ICP-06)**
    - Only if newsEvidence array is non-empty
    - Filter to evidence with BOTH source_url AND published_at < 6 months
    - If at least 1 verifiable recent news item: +10 bonus
    - If no verifiable evidence: 0 bonus (even if evidence array has items without source_url)

    **Step 6: Final score and tier (ICP-05)**
    - finalScore = clamp(0, 100, haiku_score + signalBonus + freshnessMalus + newsBonus)
    - Recalculate tier based on finalScore: >= 70 hot, >= 40 warm, < 40 cold
    - Return enriched lead with icp_score, tier, and scoring metadata (reasoning, bonuses applied)

    **Build scoring prompt function** (ICP-02):
    - `buildScoringPrompt(lead, newsEvidence, rules)`:
    - Extract positive/negative titles, target sectors, size range, min seniority from rules array
    - Format as structured French prompt asking Claude to evaluate the lead
    - Include news evidence (title + source_url) if available
    - Ask for score 0-100 and tier assignment with reasoning

    All modules use CommonJS. Error handling: if Anthropic API fails, log error and return lead as cold (fail safe — don't insert unscored leads as hot).
