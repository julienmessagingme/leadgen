<task type="auto">
  <name>Task 1: Create enrichment module with BeReach profile/company + OpenClaw Sales Nav</name>
  <files>src/lib/enrichment.js, src/lib/openclaw-browser.js</files>
  <action>
    Create `src/lib/openclaw-browser.js`:
    - Export `enrichFromSalesNav(linkedinUrl)`:
      - Call OpenClaw loopback HTTP API (localhost:18791) to navigate Sales Navigator for the given profile
      - Extract: seniority (years in role), alerts, additional data points
      - Return enrichment object or null
      - Wrap in try/catch: if OpenClaw is not running (ECONNREFUSED), log warning and return null gracefully
      - This is a nice-to-have enrichment — the pipeline must not fail if OpenClaw is unavailable
      - Note: OpenClaw has a cmdop import bug (known from Phase 1). If still broken, this module returns null and logs a warning. BeReach profile data is sufficient for scoring.

    Create `src/lib/enrichment.js`:
    - Import: bereach (visitProfile, visitCompany), openclaw-browser (enrichFromSalesNav), logger (log), supabase
    - Export `enrichLead(signal, runId)`:
      1. **Profile enrichment (ENR-01)** with 48h cache (ENR-03):
         - Check signal.profile_last_fetched_at: if exists and < 48h ago, skip BeReach call, use cached data
         - Otherwise call bereach.visitProfile(signal.linkedin_url)
         - Map response fields to lead object: first_name, last_name, headline, email, company_name, company_linkedin_url
         - Set profile_last_fetched_at = new Date().toISOString()
      2. **Company enrichment (ENR-02)**:
         - If company_linkedin_url is available, call bereach.visitCompany(company_linkedin_url)
         - Map: company_size, company_sector, company_location
         - If no company URL, skip (log info)
      3. **Sales Navigator enrichment (ENR-04)**:
         - Call enrichFromSalesNav(signal.linkedin_url)
         - If result: merge seniority data into lead object
         - If null: skip (non-critical)
      4. Return enriched lead object with all gathered data merged

    Error handling: Each enrichment step is wrapped in try/catch. If BeReach profile fails, log error and return signal with partial data. If company fails, continue. If Sales Nav fails, continue.

    All modules use CommonJS per project convention.
  </action>
  <verify>

    - `node -e "const o = require('./src/lib/openclaw-browser'); console.log(typeof o.enrichFromSalesNav);"` -> `function`
  </verify>
  <done>enrichment.js enriches leads via BeReach profile (with 48h cache), BeReach company, and optional OpenClaw Sales Nav. openclaw-browser.js handles Sales Nav with graceful fallback. Both modules deployed on VPS.</done>

<task type="auto">
  <name>Task 2: Create news evidence and FullEnrich modules</name>
  <files>src/lib/news-evidence.js, src/lib/fullenrich.js</files>
  <action>
    Create `src/lib/news-evidence.js` (ENR-05):
    - Import: supabase, logger
    - Export `gatherNewsEvidence(lead, runId)`:
      1. Skip if lead.company_name is falsy (return empty array)
      2. Build Google News RSS query: company_name + optional sector keywords, French results (hl=fr, gl=FR, ceid=FR:fr)
      3. Fetch RSS feed via Node 20 built-in fetch
      4. Parse XML with simple regex extraction (no xml2js dependency needed — Google News RSS is predictable):
         - Extract <item> blocks, then title, link, description, pubDate from each
         - Handle CDATA wrapping in title/description
      5. Take top 5 results
      6. Map to evidence objects: { lead_id, source_url, source_title, summary (max 500 chars), published_at, relevance_score: null }
      7. Insert evidence into lead_news_evidence table via Supabase
      8. Return evidence array
    - Error handling: News is non-critical. Catch all errors, log warning, return empty array.
    - Anti-hallucination: Every evidence item MUST have a source_url. Never generate news from LLM without a URL.

    Create `src/lib/fullenrich.js` (ENR-06):
    - Import: logger
    - Export `enrichContactInfo(linkedinUrl, runId)`:
      1. Submit enrichment request to FullEnrich API: POST with LinkedIn URL
      2. Poll for results (check every 30 seconds, max 10 attempts = 5 minutes)
      3. On success: return { email, phone, confidence } — only return if confidence is 'high' or 'medium'
      4. On timeout: log warning, return null
      5. On error: log warning, return null
    - Use process.env.FULLENRICH_API_KEY for auth
    - This enrichment is used BEFORE email outreach (Phase 3), but the module is created now for completeness
    - FullEnrich is async: submit then poll. Do NOT use webhook (simpler to implement polling for now).

    All modules use CommonJS per project convention.
  </action>
  <verify>
    SSH to VPS and run:
    - `node -c src/lib/news-evidence.js && node -c src/lib/fullenrich.js` -> syntax check passes
    - `node -e "const n = require('./src/lib/news-evidence'); console.log(typeof n.gatherNewsEvidence);"` -> `function`
    - `node -e "const f = require('./src/lib/fullenrich'); console.log(typeof f.enrichContactInfo);"` -> `function`
  </verify>
  <done>news-evidence.js gathers company news from Google News RSS with verifiable source URLs stored in lead_news_evidence. fullenrich.js provides async email/phone enrichment via FullEnrich API with polling. Both modules deployed on VPS.</done>
