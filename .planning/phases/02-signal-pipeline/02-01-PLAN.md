    - Uses URL constructor for parsing (not regex-only)

    - Throw descriptive errors on non-OK responses (include status code and response body)
    - Add optional delay helper: `sleep(ms)` for rate limiting between calls

    - Import @hubspot/api-client

  <files>src/lib/dedup.js</files>
  <action>
    Create `src/lib/dedup.js`:
    - Import: url-utils (canonicalizeLinkedInUrl), supabase (client), hubspot (existsInHubspot), logger (log)
    - Export `dedup(signals, runId)`:
      1. Initialize empty Set `seenUrls` for in-batch dedup
      2. For each signal in signals array:
         a. Canonicalize URL via canonicalizeLinkedInUrl -> skip if null
         b. In-batch dedup: skip if seenUrls.has(canonical), else add to set
         c. Supabase dedup (SIG-06): query leads table where linkedin_url_canonical = canonical, limit 1. Skip if found.
         d. HubSpot dedup (SIG-07): if signal has first_name and last_name, call existsInHubspot. Skip if found. Log skip reason.
         e. If passes all checks: add linkedin_url_canonical to signal object, push to unique array
      3. Return unique array

    Pattern notes:
    - Each check is sequential (canonical -> batch -> supabase -> hubspot) for correctness
    - HubSpot check only runs if name data is available (some signals may have partial data)
    - Log dedup results: how many removed at each stage
    - Error isolation: wrap individual signal processing in try/catch, log and skip on error
