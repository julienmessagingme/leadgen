**Step 2: Create src/api/dashboard.js** (CommonJS -- backend)

Create an Express router with authMiddleware applied to all routes. Three GET endpoints:

**GET /stats** -- serves DASH-01 (funnel), DASH-02 (activity), DASH-03 (LinkedIn gauge):
- Query supabase.from("leads").select("status, created_at, invitation_sent_at")
- Compute funnel counts by mapping all 12 DB statuses to 5 funnel stages:
  - "new" stage: new, enriched, scored, prospected
  - "invited" stage: invitation_sent
  - "connected" stage: connected, messaged
  - "email" stage: email_sent
  - "whatsapp" stage: whatsapp_sent, replied, meeting_booked
  - Exclude: disqualified (not in funnel)
- Compute conversion percentages between each adjacent stage (invited/new, connected/invited, etc.)
- Count leads with created_at >= todayStartParis for today count
- Count leads with created_at >= weekStartParis (Monday 00:00 Paris) for week count
- Count leads with invitation_sent_at >= todayStartParis for LinkedIn gauge
- Use Europe/Paris timezone for "today" and "this week" boundaries (reuse pattern from task-a/task-b: construct Date with toLocaleString('en-US', {timeZone: 'Europe/Paris'}))
- Return JSON: { funnel: {new, invited, connected, email, whatsapp}, conversions: {invited_pct, connected_pct, email_pct, whatsapp_pct}, activity: {today, week}, linkedin: {sent, limit: 15} }

**GET /charts** -- serves DASH-05 (sources), DASH-06 (scores), DASH-07 (trend):
- Query supabase.from("leads").select("signal_category, icp_score, created_at")
- Signal source breakdown: group by signal_category, return array of {name, value}
- ICP score histogram: bucket into 0-20, 20-40, 40-60, 60-80, 80-100 ranges, return array of {range, count}
- 7-day trend: for each of the last 7 days, count leads by created_at date (Paris timezone), return array of {date, count}
- Return JSON: { sources: [...], scores: [...], trend: [...] }

**GET /cron** -- serves DASH-04:
- Define task names: task-a-signals, task-b-invitations, task-c-followup, task-d-email, task-e-whatsapp, task-f-briefing
- For each task, query supabase.from("logs").select("task, level, message, created_at").eq("task", taskName).order("created_at", {ascending: false}).limit(1) to get the latest log entry
- Determine status from latest entry: if message includes "completed" -> "ok", "error" -> "error", "started" -> "running", no entry -> "never"
- IMPORTANT: Read the actual logger.js on VPS first (`cat /home/openclaw/leadgen/src/lib/logger.js`) to confirm exact message format before coding the query. Adapt the query pattern to match whatever format logTaskRun actually produces.
- Return JSON: { tasks: [{task, label, status, lastRun}] }
- Include French labels in response: "A - Signaux", "B - Invitations", "C - Follow-up", "D - Email", "E - WhatsApp", "F - Briefing"
