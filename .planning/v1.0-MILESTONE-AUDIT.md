## Integration Checker Findings (Critical)

### [P0] icp-scorer.js — Beta API Path Not Applied on VPS

**Severity:** Critical — blocks entire signal-to-lead pipeline
**Impact:** ICP-01, ICP-05, SIG-01 through SIG-04 (leads collected but all score cold and get dropped)
**Root Cause:** Commit `1f91444` (beta API fix from plan 02-05) does not exist in VPS git repo. The live file still uses `getAnthropicClient().messages.create` instead of `getAnthropicClient().beta.messages.create`. The `output_config` json_schema is present but rejected by standard API path.
**Fix:** On VPS, edit `/home/openclaw/leadgen/src/lib/icp-scorer.js` line 127: `messages.create` → `beta.messages.create`. Commit and restart PM2.

### [P0] Missing Database Columns — follow_up_sent_at + whatsapp_template_created_at

**Severity:** Critical — breaks Task C idempotency and Task E template tracking
**Impact:** LIN-07 (follow-up reprocessing), WA-01 (duplicate WhatsApp templates)
**Root Cause:** Columns used by Phase 3 tasks were never added to the Supabase schema (001_initial_schema.sql)
**Fix:** Run migration:
```sql
ALTER TABLE leads ADD COLUMN IF NOT EXISTS follow_up_sent_at timestamptz;
ALTER TABLE leads ADD COLUMN IF NOT EXISTS whatsapp_template_created_at timestamptz;
```
