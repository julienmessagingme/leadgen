  <action>
Create `src/lib/sales-nav-scraper.js` (CommonJS module):

**Exports:** `searchSalesNav(filters, runId)` where filters = {sector, company_size, job_title, geography, max_leads}

**Implementation:**

1. **Build Sales Nav search URL:**
   - Base: `https://www.linkedin.com/sales/search/people`
   - Map form fields to Sales Nav URL query parameters:
     - job_title -> `keywords` parameter (titleIncluded)
     - sector -> `industryIncluded` parameter
     - company_size -> `companySizeIncluded` parameter (map ranges: "1-10" -> "B", "11-50" -> "C", "51-200" -> "D", "201-500" -> "E", "501-1000" -> "F", "1000+" -> "G")
     - geography -> `geoIncluded` parameter
   - NOTE: Sales Nav uses encoded filter blobs in the URL. The simplest approach is to navigate to the search page and use the UI filter sidebar OR build the URL with `keywords` query param for job title and apply other filters via page interaction. Choose the approach that is most resilient. Document the chosen approach clearly.

2. **Navigate and scrape:**
   - Use createBrowserContext() from browser.js
   - Use navigateWithLimits() for all navigation (respects 100/day limit)
   - Use humanDelay() between actions (3-8s random)
   - Navigate to Sales Nav search URL
   - Wait for results list to load (selector: `.search-results__result-list` or similar, with multi-selector fallback pattern from Phase 12)
   - Call dismissPopups(page) if available from browser-signal-collector, otherwise implement inline

3. **Extract profiles from results list:**
   - For each result card on the page, extract:
     - first_name, last_name (from name element, split on space)
     - headline (from subtitle/headline element)
     - company_name (from company link/text)
     - linkedin_url (from profile link href, convert sales nav URL to regular linkedin.com/in/ URL)
   - Use multi-selector fallback strategy (try 3+ selector variants for each field)
   - If a field cannot be extracted, set null (do not crash)

4. **Pagination (if needed to reach max_leads):**
   - Sales Nav shows ~25 results per page
   - If max_leads > 25, navigate to page 2 using next button or page parameter
   - Maximum 2 pages (50 results cap per CONTEXT decision)
   - Use navigateWithLimits + humanDelay for each page navigation

5. **Error handling:**
   - If CAPTCHA detected (page contains captcha-related selectors or text): stop immediately, log warning, return partial results
   - If session expired (redirect to login): stop, log error, return empty array
   - If page structure unexpected (no results found): log warning, return empty array
   - Always close browser in finally block via closeBrowser()
   - Send email alert to Julien via gmail.js on CAPTCHA or session expiry (same pattern as 12-03 cookie expiry alert)

6. **Return value:**
   ```js
   {
     profiles: [{ first_name, last_name, headline, company_name, linkedin_url }],
     pages_consumed: Number,  // for page budget tracking
     stopped_reason: null | 'captcha' | 'session_expired' | 'max_reached' | 'no_results'
   }
   ```

**Anti-detection (use Phase 11/12 patterns):**
- Chrome 120 UA, fr-FR locale already set by createBrowserContext
- humanDelay() between every action (scroll, click, navigation)
- Random scroll behavior before extraction (scroll down slowly, wait, scroll back)
