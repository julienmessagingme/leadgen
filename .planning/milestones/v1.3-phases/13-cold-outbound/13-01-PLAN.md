1. Create migration `src/db/migrations/009-cold-searches-table.sql`:
   - Table `cold_searches` with columns:
     - id UUID DEFAULT gen_random_uuid() PRIMARY KEY
     - filters JSONB NOT NULL (stores {sector, company_size, job_title, geography, max_leads})
     - status TEXT DEFAULT 'pending' (pending, running, completed, error)
     - leads_found INTEGER DEFAULT 0
     - leads_enriched INTEGER DEFAULT 0
     - error_message TEXT
     - created_at TIMESTAMPTZ DEFAULT now()
     - completed_at TIMESTAMPTZ
   - Index on created_at DESC for history listing
   - RLS policy for authenticated role (same pattern as other tables)

2. Create `src/api/cold-outbound.js` Express router (CommonJS, same pattern as src/api/settings.js):
   - Apply authMiddleware via router.use() at top
   - POST /search: validates body {sector, company_size, job_title, geography, max_leads}
     - max_leads must be 1-50 (reject otherwise with 400)
     - sector and job_title required (non-empty strings)
     - Insert into cold_searches with status 'pending', return the search record with 201
     - NOTE: Actual scraping will be triggered by plan 13-03 which will add the execution logic
   - GET /searches: list all cold_searches ordered by created_at DESC, limit 50
   - GET /searches/:id: get single search with its leads (join leads where metadata->>search_id = id)
   - GET /searches/:id/status: lightweight polling endpoint returning {id, status, leads_found, leads_enriched}

3. Mount in src/index.js:
   - Add `app.use("/api/cold-outbound", require("./api/cold-outbound"));` after settings route, before static serve

Run the migration on VPS via SSH: `ssh ubuntu@146.59.233.252 "cd /home/openclaw/leadgen && node -e \"require('dotenv').config(); const {supabase}=require('./src/lib/supabase'); const fs=require('fs'); supabase.rpc('exec_sql',{sql: fs.readFileSync('src/db/migrations/009-cold-searches-table.sql','utf8')}).then(r=>console.log(r))\""`
