  <name>Task 1: Create browser-signal-collector.js with competitor_page and influencer scrapers</name>
  <files>src/lib/browser-signal-collector.js</files>
  <action>
Create src/lib/browser-signal-collector.js with a collectBrowserPageSignals(runId) function that:

1. Loads active watchlist entries from Supabase where source_type IN ('competitor_page', 'influencer') and is_active = true.

2. Creates a browser context via createBrowserContext(runId) from src/lib/browser.js. If browser creation fails (cookies expired), log error and return empty array (do NOT throw -- let Task A continue).

3. For each watchlist source:
   a. Navigate to the source's LinkedIn page (source_url) using navigateWithLimits(page, url). The source_url is a LinkedIn company/person page URL.
   b. Wait for the page to load, then look for recent posts on the page. LinkedIn pages show recent activity. Find post containers (CSS selectors for feed posts on a profile/company page). Take the first 2-3 recent posts visible.
   c. For each post, click the reactions count element (the "X reactions" or "X likes" link/button) to open the reactions popup/modal.
   d. Wait for the popup to appear (modal with list of people who reacted).
   e. Extract profiles from the FIRST SCREEN ONLY (no scrolling per user decision): for each visible entry in the popup, extract:
      - Name (full name text)
      - Headline (text below the name)
      - Profile URL (href of the link to their profile)
   f. Close the popup.
   g. Use humanDelay() between each action (clicking, reading popup).
   h. If a post has 0 reactions or the popup fails to open, log debug and skip silently.

4. Format collected profiles into signal objects matching the format used by signal-collector.js formatSignals:
   - linkedin_url: extracted profile URL
   - first_name: split from full name (first word)
   - last_name: split from full name (remaining words)
   - headline: extracted headline
   - company_name: null (not reliably available from popup)
   - signal_type: "like" (we're getting reactions, close enough)
   - signal_category: "concurrent" for competitor_page, "influenceur" for influencer (matching Bereach conventions)
   - signal_source: source.source_label || source.source_type
   - signal_date: new Date().toISOString()
   - sequence_id: source.sequence_id || null
   - source_origin: "browser" (NEW FIELD for BSIG-06 source tagging)

5. Wrap each source in try/catch for error isolation (one failing source does not crash collection). If navigateWithLimits throws (rate limit reached), log warning, keep partial results, and stop processing remaining sources.

6. Close browser at the end (closeBrowser) in a finally block.

7. Log summary: number of leads collected per source, total, pages consumed.

8. Handle LinkedIn popups/modals that may appear (cookie consent, "Sign in to continue"): dismiss them by clicking close/accept buttons if detected. Add a helper function dismissPopups(page) that checks for common overlay selectors and dismisses them.

Important CSS selector notes for LinkedIn (use data attributes and aria labels where possible, class names change frequently):
- Reactions count on a post: look for button/span with text matching pattern like "\\d+ reaction" or aria-label containing "reaction"
- Reactions modal: look for a modal/dialog overlay that appears after clicking reactions
- Profile entries in modal: each entry typically has a link to the profile, a name element, and a description/headline element
- Use page.locator() with flexible selectors, add fallback selectors

CommonJS module pattern. Export { collectBrowserPageSignals }.
