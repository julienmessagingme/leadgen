  <name>Task 1: Add source_origin tagging and orchestrating function</name>
  <files>src/lib/signal-collector.js, src/lib/browser-signal-collector.js</files>
  <action>
Part A - Tag Bereach signals with source_origin in signal-collector.js:

1. In the formatSignals function in signal-collector.js, add source_origin: "bereach" to the returned signal object. This ensures every Bereach-collected signal carries its origin.

Part B - Create orchestrating function in browser-signal-collector.js:

2. Add collectAllBrowserSignals(runId) function that:
   a. Creates browser context via createBrowserContext(runId). If creation fails (cookies expired):
      - Send email alert to Julien using sendEmail from gmail.js: subject "LinkedIn cookies expires - browser scraping desactive", body explaining cookies need renewal with instructions.
      - Log error and return { signals: [], stats: { error: "cookies_expired" } }
      - Do NOT throw.
   b. Opens a page from the browser context.
   c. Calls collectBrowserPageSignals(page, runId) -- competitor_page + influencer signals
   d. Calls collectBrowserKeywordSignals(page, runId) -- keyword signals
   e. Calls collectBrowserJobSignals(page, runId) -- job_keyword signals
   f. Each call wrapped in try/catch. If one fails, log and continue with others.
   g. Merge all signals into a single array.
   h. Close browser in finally block.
   i. Return { signals: allSignals, stats: { competitor_page: N, influencer: N, keyword: N, job_keyword: N, pages_consumed: getPageCount().count, errors: N } }

3. Refactor the three scraper functions (from plans 01 and 02) to accept (page, runId) as parameters instead of creating their own browser context. They should receive the page from collectAllBrowserSignals. Also update collectBrowserPageSignals to no longer create/close browser internally -- it receives the page.

4. Export collectAllBrowserSignals from the module.

Note on dedup (BSIG-05): The existing dedup.js already checks Supabase for linkedin_url_canonical. Since Bereach runs first and inserts leads into Supabase, when browser signals go through the same dedup pipeline, any lead already inserted by Bereach will be automatically skipped. This is permanent dedup (not just same-day). No additional dedup logic needed -- the existing pipeline handles it. The source_origin field distinguishes the source after the fact.
