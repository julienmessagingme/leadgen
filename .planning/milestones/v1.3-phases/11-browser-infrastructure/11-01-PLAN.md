<task type="auto">
  <name>Task 1: Installer Playwright et creer le module browser.js</name>
  <files>package.json, src/lib/browser.js, linkedin-cookies.json</files>
  <action>
1. Ajouter playwright comme dependance dans package.json et lancer `npm install playwright` puis `npx playwright install chromium --with-deps` sur le VPS via SSH (ubuntu@146.59.233.252). Le install --with-deps installe les libs systeme necessaires pour Chromium headless sur Ubuntu.

2. Creer `src/lib/browser.js` avec les exports suivants :

- `createBrowserContext()` : Lance chromium headless via `chromium.launch({ headless: true })`, cree un BrowserContext, charge les cookies depuis `linkedin-cookies.json` (chemin relatif au root projet, configurable via env `LINKEDIN_COOKIES_PATH`). Avant de naviguer, appelle `validateSession()`. Si la session est invalide, log une alerte via le logger existant (`require('./logger').log`) avec level 'error' et message "LinkedIn session expired - browser actions blocked", puis throw une Error. Si valide, retourne { browser, context, page }.

- `validateSession(context)` : Navigue vers `https://www.linkedin.com/feed/` avec le context, attend le chargement, verifie que l'URL finale ne contient pas `/login` et que la page contient un element `.feed-shared-update-v2` ou `[data-urn]` (indicateurs du feed). Retourne true/false. Timeout de 15 secondes.

- `closeBrowser(browser)` : Ferme le browser proprement avec `browser.close()`.

- Pattern anti-detection minimal : user-agent realiste (Chrome 120+ sur Linux), viewport 1920x1080, locale fr-FR, timezone Europe/Paris. Pas de stealth plugin pour l'instant (volume < 100/jour).

3. Creer `linkedin-cookies.json` comme template avec un tableau vide `[]` et un commentaire dans un README ou en haut du fichier expliquant le format attendu :
```json
[
  {"name": "li_at", "value": "REPLACE_ME", "domain": ".linkedin.com", "path": "/"},
  {"name": "JSESSIONID", "value": "REPLACE_ME", "domain": ".linkedin.com", "path": "/"}
]
```
Le cookie `li_at` est le cookie de session principal LinkedIn. `JSESSIONID` est aussi necessaire. Format Playwright : tableau d'objets avec name, value, domain, path.

4. Ajouter `linkedin-cookies.json` dans `.gitignore` (contient des secrets). Le template commite doit avoir des valeurs REPLACE_ME, pas de vrais cookies.

Important : utiliser le meme pattern que les autres modules lib (require, module.exports, JSDoc comments, logger). Ne PAS utiliser ES modules (import/export). Le projet est en CommonJS.
