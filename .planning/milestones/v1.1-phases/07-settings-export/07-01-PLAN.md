**Settings router endpoints (all behind authMiddleware):**

CONF-01 - ICP Rules:
- `GET /icp-rules` -- `supabase.from('icp_rules').select('*').order('category')` returns `{ rules: data }`
- `POST /icp-rules` -- validates `category` against VALID_ICP_CATEGORIES array (`title_positive`, `title_negative`, `sector`, `company_size`, `seniority`, `freshness`, `signal_weights`), inserts `{ category, value, key, numeric_value, threshold }`, returns created row
- `PUT /icp-rules/:id` -- updates row by id with same fields, returns updated row
- `DELETE /icp-rules/:id` -- deletes by id, returns `{ ok: true }`

CONF-02 - Suppression List:
- `GET /suppression` -- `supabase.from('suppression_list').select('*')` returns `{ entries: data }` (hashed_value + source columns)
- `POST /suppression` -- accepts `{ value, source }` where source is 'email' or 'linkedin'. Hash `value` server-side with `crypto.createHash('sha256').update(value.trim().toLowerCase()).digest('hex')` (same pattern as leads.js). Insert `{ hashed_value, source }`. Return `{ ok: true }`.
- `DELETE /suppression/:hash` -- deletes by hashed_value, returns `{ ok: true }`

CONF-03 + CONF-05 - Config (limits + templates):
- `GET /config` -- `supabase.from('settings').select('*')` returns `{ settings: data }` as key-value pairs
- `PATCH /config/:key` -- accepts `{ value }`, upserts into settings table with `updated_at = now()`, returns updated row

CONF-04 - Watchlist:
- `GET /watchlist` -- `supabase.from('watchlist').select('*').order('source_type')` returns `{ sources: data }`
- `POST /watchlist` -- inserts `{ source_type, source_label, source_url, keywords, is_active, sequence_id }`, returns created row
- `PUT /watchlist/:id` -- updates row by id, returns updated row
- `DELETE /watchlist/:id` -- deletes by id, returns `{ ok: true }`

**CSV Export (in src/api/leads.js):**

Add `GET /export` endpoint BEFORE the `GET /:id` route (to avoid Express treating "export" as an :id param). This endpoint:
1. Reads query params: `status`, `tier`, `source`, `search`, `date_from`, `date_to`
2. Builds Supabase query on `leads` table selecting: `first_name, last_name, email, linkedin_url, icp_score, tier, status, company_name, created_at`
3. Applies same filter logic as the existing list endpoint (status split by comma with `.in()`, tier `.eq()`, source `.eq('signal_category')`, search with `.or()` on first_name/last_name/company_name)
4. Adds date range filters: `if (date_from) query = query.gte('created_at', date_from)` and `if (date_to) query = query.lte('created_at', date_to)`
5. Adds `.limit(10000)` to prevent timeout on large datasets
6. Orders by `created_at` desc
7. Builds CSV with French headers: `Nom,Prenom,Email,LinkedIn,Entreprise,Score ICP,Tier,Statut,Date`
8. Properly escapes values: `'"' + String(v).replace(/"/g, '""') + '"'`
9. Sets response headers:
   - `Content-Type: text/csv; charset=utf-8`
   - `Content-Disposition: attachment; filename="leads-export.csv"`
10. Prepends BOM `\uFEFF` for Excel UTF-8 compatibility
11. Sends CSV string
